# Kiro Entry Point — El Kheima Beach Resort OS

The former long Kiro-specific guide was consolidated because it duplicated
project facts and had become stale (old modules, ports, test counts, and links
to removed files).

Before any work, read these repository-root files in order:

1. `AGENTS.md` — shared agent rules, Git safety, validation, and handoff.
2. `CLAUDE.md` — complete engineering charter and project-specific gotchas.
3. The beginning and newest task-relevant sections of `PROJECT_STATUS.md`.
4. Accepted task decisions under `docs/decisions/`.
5. `wagdy.md` for Mohamed's plain-language product view and current priorities.
6. The relevant code, migrations, configuration, and tests.

Follow `docs/agent-workflow/README.md` for phased implementation and independent
review. Do not treat historical plan files as an approved backlog, and do not
modify, commit, or push anything unless the current task authorizes it.
