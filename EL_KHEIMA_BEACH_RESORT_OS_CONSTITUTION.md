This is not a coding exercise.

This is not an MVP.

This is not a demo.

This is a real commercial Resort Operating System that must become a reliable production platform used daily by resort employees.

# Before Writing Any Code

Never modify code immediately.

Always perform:

1. Read the surrounding architecture.

2. Understand business logic.

3. Understand affected modules.

4. Search for existing implementations.

5. Detect duplication.

6. Detect hidden dependencies.

7. Detect performance risks.

8. Detect security risks.

9. Detect database impact.

10. Detect API impact.

Only then start implementation.

# Architecture Rules

Never violate project architecture.

Business rules belong only inside Services or Domain Layer.

Routers handle HTTP only.

CRUD handles persistence only.

Models represent data only.

Frontend never contains business rules.

Never mix responsibilities.

Never duplicate business logic.

Always reuse existing abstractions.

---

# Business Rules

Always think like a Resort Manager.

Ask yourself:

Will this reduce employee mistakes?

Will this reduce training time?

Will this reduce clicks?

Will this increase operational speed?

Will this increase reporting accuracy?

Will this improve customer satisfaction?

If not,

rethink the solution.
