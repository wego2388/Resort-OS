## أول يوم — جميع الأدوار

1. افتح رابط التطبيق وأدخل بريدك الإلكتروني وكلمة المرور المؤقتة.
2. ستُجبَر على تغيير كلمة المرور فورًا — اختر كلمة تتذكرها، 8 أحرف كحد أدنى.
3. **للمحاسبين ومديري الموارد البشرية والمديرين:** بعد تغيير كلمة المرور
   ستنتقل لشاشة إعداد المصادقة الثنائية. امسح رمز QR بتطبيق Google
   Authenticator أو Authy. دوّن رموز الاسترداد واحتفظ بها بأمان.
   ستحتاج رمزًا مكوّنًا من 6 أرقام من التطبيق عند كل تسجيل دخول.
4. بعد الإعداد ستنتقل لشاشتك الرئيسية تلقائيًا.

## المدير / admin / manager

**الشاشة الرئيسية:** `/admin/dashboard`

### المهام اليومية

- **الصباح:** لوحة التحكم ← إيرادات اليوم، الإشغال، التنبيهات.
- **أثناء اليوم:** الموافقة على طلبات الشراء المعلّقة، مراجعة أوامر الصيانة.
- **نهاية اليوم:** تقارير الوردية (المالية ← الورديات)، مراجعة سجل التدقيق.

### شاشاتك

| الشاشة | المسار | ماذا تفعل هناك |
|---|---|---|
| لوحة التحكم | الإدارة ← لوحة التحكم | الإيرادات الحية، الإشغال، المؤشرات |
| التحليلات | الإدارة ← التحليلات | الاتجاهات، مخططات الإشغال، الطاقة |
| المالية | الإدارة ← المالية | الحسابات، القيود اليومية |
| الورديات | الإدارة ← المالية ← الورديات | فتح/إغلاق ورديات الكاشير، التقارير |
| الموارد البشرية | الإدارة ← الموارد البشرية | الموظفون، الرواتب، الحضور |
| قائمة الدايننج | الإدارة ← قائمة الدايننج | المنافذ، الفئات، الأصناف، الطاولات |
| الوصفات | الإدارة ← الوصفات | وصفات الأصناف + تكلفة الطعام |
| تكلفة الطعام | الإدارة ← تكلفة الطعام | تقرير التكلفة حسب المنفذ/الفترة |
| المخزون | الإدارة ← المخزون | المخزون، المنتجات، الموردون |
| إدارة الشاطئ | الإدارة ← إدارة الشاطئ | مواقع الشاطئ، عقود B2B |
| إدارة العملاء | الإدارة ← CRM | العملاء، العملاء المحتملون، الحملات |
| التايم شير | الإدارة ← التايم شير | العقود، الأقساط، الزيارات |
| الصيانة | الإدارة ← الصيانة | الأصول، أوامر العمل، الجداول |
| الإيجارات | الإدارة ← الإيجارات | عقود الإيجار، المدفوعات |
| الإعدادات | الإدارة ← الإعدادات | إعدادات الفرع |
| الصلاحيات | الإدارة ← الصلاحيات | منح/حجب صلاحيات محددة |
| سجل التدقيق | الإدارة ← Super Admin ← التدقيق | جميع الأحداث الحساسة |
| رموز QR | الإدارة ← QR Codes | توليد رموز QR للطاولات والمواقع |
| الموقع والحجوزات | الإدارة ← الموقع | إدارة محتوى الموقع، الحجوزات الأونلاين |

### إجراءات تحتاج موافقة مدير / super_admin

- إلغاء دفعة مُسجَّلة (المالية)
- تطبيق خصم فوق الحد المحدد للمنفذ
- إلغاء حجز بعد تسجيل الوصول
- الموافقة على طلبات الشراء
- الموافقة على طلبات الإجازة
- الموافقة على مسيرات الرواتب
- الإعدادات العامة (النظام كله) — تتطلب مصادقة Step-Up من super_admin

## المحاسب / accountant

### المهام اليومية

- مراجعة المالية ← الورديات للتسويات المكتملة.

- مراجعة المالية ← الفواتير للأرصدة المعلّقة للضيوف.
- متابعة قائمة انتظار فواتير ETA الإلكترونية.

| الشاشة | المسار | ماذا تفعل هناك |
|---|---|---|
| المالية | الإدارة ← المالية | الحسابات، القيود، أسعار الصرف |
| الفواتير (الفوليو) | الإدارة ← المالية ← الفواتير | فواتير الضيوف |
| مراكز التكلفة | الإدارة ← المالية ← مراكز التكلفة | توزيع التكاليف |
| تقارير الورديات | الإدارة ← المالية ← الورديات | تقارير تسوية الورديات |
| الميزانية | الإدارة ← المالية ← التقارير | ميزان المراجعة، قائمة الدخل |
| الفاتورة الإلكترونية | الإدارة ← الفاتورة الإلكترونية | فواتير هيئة الضرائب المصرية |
| الاستهلاك | الإدارة ← المالية ← الاستهلاك | استهلاك الأصول الثابتة |
| الحسابات البنكية | الإدارة ← المالية ← الحسابات البنكية | مطابقة الحسابات |
| الخصومات | الإدارة ← المالية ← الخصومات | إعداد قواعد الخصم |

### إجراءات تحتاج موافقة المدير

- إغلاق فترة محاسبية (المالية ← الفترات)
- إلغاء دفعة مُسجَّلة

### تنبيه مهم

- المصادقة الثنائية إلزامية لك. إذا تخطّيت الإعداد، ستظهر الشاشات فارغة
  وكل استدعاء للـ API سيفشل بـ 403 — هذا متوقع، أكمل إعداد 2FA.
- فواتير ETA: يُنشئها النظام تلقائيًا بعد الدفع. مهمتك متابعة الحالة
  ومعالجة المرفوضات.

## الاستقبال / receptionist

**الشاشة الرئيسية:** `/ops/reception`

### المهام اليومية

1. فحص قائمة الوصول لهذا اليوم — العمليات ← الحجوزات ← فلتر بتاريخ تسجيل
   الوصول اليوم.
2. تسجيل وصول الضيوف القادمين — ابحث عن الحجز ← انقر تسجيل الوصول ← أكّد.
3. تسجيل مغادرة الضيوف المغادرين — ابحث عن الحجز ← تسجيل المغادرة ← سدّد
   الفاتورة إذا كان هناك رصيد معلّق.
4. معالجة تعيين الغرف وطلبات تسجيل الوصول المبكر/المتأخر.
5. نهاية اليوم: أبلغ المدير عن الغائبين بدون إشعار.

### شاشاتك

| الشاشة | المسار | ماذا تفعل هناك |
|---|---|---|
| الاستقبال | العمليات ← الاستقبال | لوحة تسجيل الوصول/المغادرة |
| الحجوزات | العمليات ← الحجوزات | جميع الحجوزات، إنشاء حجز جديد |
| الغرف | العمليات ← الغرف | توافر الغرف وحالتها |
| التدبير المنزلي | العمليات ← التدبير المنزلي | الإبلاغ عن مشاكل الغرف |
| فاتورة الضيف | من الحجز ← الفاتورة | الرسوم والمدفوعات للضيف |

### إجراءات تحتاج موافقة المدير

- منح غرفة مجانية (تسعير صفري على الفاتورة)
- إلغاء حجز مع التنازل عن الغرامة
- تسجيل وصول بدون حجز مسبق (غير اعتيادي)

### مواقف شائعة

| الموقف | ماذا تفعل |
|---|---|
| الضيف سجّل وصوله في غرفة خاطئة | العمليات ← الغرف ← نقل الغرفة — اتصل بالمدير أولًا |
| الضيف يطلب تسجيل وصول مبكر | العمليات ← الحجوزات ← الحجز ← علامة مبكر/متأخر |
| رصيد معلّق عند المغادرة | المالية ← الفواتير ← سدّد قبل إتمام تسجيل المغادرة |
| الغرفة تبدو مشغولة في النظام لكن الضيف غادر | العمليات ← الغرف ← علّمها نظيفة يدويًا — أبلغ المشرف |

## الكاشير / cashier

**الشاشة الرئيسية:** `/pos/beach`

### المهام اليومية

1. **بداية الوردية:** نقطة البيع ← الوردية ← فتح الوردية. أدخل مبلغ النقدية الافتتاحي.
2. **أثناء الوردية:** معالجة معاملات الشاطئ، إصدار الإيصالات.
3. **نهاية الوردية:** نقطة البيع ← الوردية ← إغلاق الوردية. عدّ النقدية،
   أدخل المبالغ الختامية لكل عملة، أرسل التقرير.

### شاشاتك

| الشاشة | المسار | ماذا تفعل هناك |
|---|---|---|
| بيع الشاطئ | نقطة البيع ← الشاطئ | بيع خدمات الشاطئ، المظلات، التذاكر |
| خريطة الشاطئ | نقطة البيع ← خريطة الشاطئ | الخريطة الحية للمواقع |
| بيع الدايننج | نقطة البيع ← الدايننج | طلبات الطاولات (إذا كنت مُعيَّنًا) |
| لوحة الوردية | نقطة البيع ← الوردية | فتح/إغلاق الوردية، حركات النقدية |

### إجراءات تحتاج موافقة المدير/المشرف

- **إلغاء معاملة** بعد ترحيلها — امسح QR أو اتصل بالمدير لموافقة PIN
- تطبيق خصم غير موجود في قائمة الأسعار القياسية
- حركات النقدية (إضافة/سحب نقدية من الدرج خلال الوردية) — أدخل السبب،
  يُبلَّغ المدير تلقائيًا

### نصائح

- عملاء B2B (شركات): اختر تسجيل دخول B2B. النظام يفحص حصة العقد تلقائيًا.
- وضع عدم الاتصال: إذا انقطع الاتصال، تُخزَّن المعاملات محليًا وتتزامن عند
  عودة الاتصال. ستظهر أيقونة تزامن برتقالية.
- **لا تُغلق تبويب الوردية دون إتمام عدّ النقدية.** الوردية غير المكتملة
  لا يمكن إغلاقها عن بُعد.

## النادل / waiter

**الشاشة الرئيسية:** `/pos/dining`

### المهام اليومية

1. افتح نقطة بيع الدايننج.
2. اختر طاولة أو أنشئ طلبًا للتيك أواي.
3. أضف الأصناف ← أرسل للمطبخ (شاشة KDS في المطبخ تتحدث تلقائيًا).
4. عند رغبة الضيف بالدفع، سلّم للكاشير أو اعالج الدفع مباشرةً إذا كنت
   تملك صلاحيات كاشير.

### شاشاتك

| الشاشة | المسار | ماذا تفعل هناك |
|---|---|---|
| دايننج POS | نقطة البيع ← الدايننج | الطاولات، الطلبات، الإرسال للمطبخ |
| شاشة المطبخ (للمتابعة) | KDS ← الدايننج | متابعة حالة تذاكر المطبخ |

### إجراءات تحتاج موافقة المدير

- **إلغاء صنف** بعد إرساله للمطبخ — يتطلب PIN المدير
- تطبيق خصم فوق حد المنفذ
- تقسيم فاتورة بعد ترحيل الدفع

### نصائح

- الإضافات والمُعدِّلات: عند إضافة صنف تظهر نافذة للإضافات (مثلًا: بدون
  بصل، صوص إضافي). اختر قبل الإرسال.
- الطلبات الموقوفة: يمكنك إيقاف طلب مؤقتًا (حفظ دون إرسال) إذا لم يكن
  الضيف جاهزًا.
- نقل الطاولة: دايننج POS ← الطلب ← نقل — ينقل جميع الأصناف لطاولة أخرى.
- تنبيهات الضيوف: الضيوف على الطاولات المزوّدة بـ QR يمكنهم طلب الجرسون
  أو الفاتورة — يظهر إشعار على شاشتك.

## المطبخ / chef / kitchen

**الشاشة الرئيسية:** `/kds/dining`

### المهام اليومية

1. شاشة KDS تُحمَّل تلقائيًا وتُظهر جميع التذاكر النشطة.
2. عند البدء في تحضير تذكرة: انقر ← **قيد التحضير**.
3. عند الانتهاء: انقر ← **مكتمل**.
4. الأصناف المكتملة تختفي من قائمة الانتظار.

### شاشاتك

| الشاشة | المسار | ماذا تفعل هناك |
|---|---|---|
| شاشة المطبخ (KDS) | KDS ← الدايننج | جميع تذاكر المطبخ النشطة |

### الفلترة حسب المحطة

تُظهر الشاشة جميع المحطات افتراضيًا. لرؤية محطتك فقط (ساخن، بار،
بارد، شواية، حلويات) استخدم تبويبات الفلتر في الأعلى.

### إجراءات تحتاج موافقة المدير

- لا تُنشئ أي موافقات. إذا أُلغي صنف بعد الإرسال، النادل أو المدير
  يتولى ذلك.

### نصائح

- الألوان: التذاكر الجديدة بيضاء، التذاكر التي انتظرت أكثر من X دقيقة تصبح
  صفراء ثم حمراء. الحد الزمني يضبطه مديرك.
- إذا تجمّدت الشاشة: حدّث المتصفح (F5). التذاكر على السيرفر — لا يضيع شيء.

### المهام اليومية

- فحص تنبيهات نقص المخزون (المخزون ← مخزون منخفض).
- استلام البضائع الموصّلة مقابل أوامر الشراء المفتوحة.
- إنشاء طلبات شراء للأصناف التي تحتاج طلبًا.

### شاشاتك

| الشاشة | المسار | ماذا تفعل هناك |
|---|---|---|
| المنتجات | الإدارة ← المخزون ← المنتجات | كتالوج المنتجات، مستويات المخزون |
| حركات المخزون | الإدارة ← المخزون ← الحركات | معاملات الدخول/الخروج |
| طلبات الشراء | الإدارة ← المخزون ← طلبات الشراء | طلب أصناف للموافقة |
| أوامر الشراء | الإدارة ← المخزون ← أوامر الشراء | الطلبات المؤكّدة للموردين |
| الجرد | الإدارة ← المخزون ← الجرد | مطابقة الجرد الفعلي |
| الموردون | الإدارة ← المخزون ← الموردون | قائمة الموردين وبياناتهم |
| الفئات | الإدارة ← المخزون ← الفئات | تصنيفات المنتجات |
| المستودعات | الإدارة ← المخزون ← المستودعات | مواقع التخزين |
| تكلفة الطعام | الإدارة ← تكلفة الطعام | تقرير التكلفة المرتبط بالوصفات |
| الوصفات | الإدارة ← الوصفات | قوائم مكونات الأصناف + التكاليف |

### إجراءات تحتاج موافقة المدير

- **الموافقة على طلب الشراء** ← المدير يحوّله لأمر شراء
- **إرسال الجرد** ← المدير يوافق على الفروق قبل تعديل المخزون
- شطب بضائع تالفة ← أبلغ المدير الذي يُنشئ القيد المحاسبي

### سير عمل: طلب مخزون جديد

1. المخزون ← طلبات الشراء ← طلب جديد.
2. أضف الأصناف + الكميات + المورد المفضّل.
3. أرسل — يصل المدير إشعار للموافقة.
4. عند الموافقة، يُنشأ أمر الشراء تلقائيًا.
5. عند وصول البضائع، افتح الأمر ← علّم الأصناف مستلمة ← يتحدث المخزون.

### الوصفات وتكلفة الطعام

- الإدارة ← الوصفات: اربط المكونات (منتجات المخزون) بأصناف القائمة.
- كل مكوّن بكميته وتكلفته. النظام يحسب تكلفة الطعام النظرية تلقائيًا.
- الإدارة ← تكلفة الطعام: قارن التكلفة النظرية بالفعلية لكل منفذ/فترة.

## وكيل التايم شير / timeshare\_agent

**الشاشة الرئيسية:** `/admin/timeshare`

### المهام اليومية

- مراجعة الزيارات القادمة (التايم شير ← الزيارات القادمة).
- معالجة مدفوعات الأقساط المستحقة لهذا اليوم.
- تحديث حالة الزيارة بعد تسجيل وصول الضيف.

### شاشاتك

| الشاشة | المسار | ماذا تفعل هناك |
|---|---|---|
| العقود | الإدارة ← التايم شير ← العقود | جميع عقود التايم شير |
| الأقساط | الإدارة ← التايم شير ← الأقساط | جدول السداد، تحصيل الدفعات |
| الزيارات | الإدارة ← التايم شير ← الزيارات | حجز وإدارة زيارات الوحدات |
| الوحدات | الإدارة ← التايم شير ← الوحدات | أنواع الوحدات المتاحة والأسابيع |
| قائمة الانتظار | الإدارة ← التايم شير ← قائمة الانتظار | الضيوف المنتظرون لأسابيع محددة |
| لوحة المبيعات | الإدارة ← التايم شير ← لوحة المبيعات | خط الأنابيب والأداء |
| التقويم | الإدارة ← التايم شير ← التقويم | توافر الأسابيع بصريًا |
| إدارة العملاء | الإدارة ← CRM | ملفات العملاء، العملاء المحتملون |

### إجراءات تحتاج موافقة المدير

- إلغاء عقد
- التنازل عن غرامة التأخر في السداد
- نقل وحدة بين عقدين

### نصائح

- مدفوعات الأقساط: التايم شير ← الأقساط ← ابحث عن البند المستحق ← ادفع.
  اختر طريقة الدفع (نقدًا / بطاقة / تحويل).
- بعد التحصيل، يتحدث الفوليو تلقائيًا.
- الاستبيان: بعد اكتمال الزيارة يمكن إرسال استبيان واتساب تلقائيًا إذا
  كان مُعدًّا. يمكنك أيضًا إرساله يدويًا من الزيارات ← إرسال الاستبيان.

## الموظف / employee

**الشاشة الرئيسية:** `/portal/attendance`

### شاشاتك

| الشاشة | المسار | ماذا تفعل هناك |
|---|---|---|
| الحضور | البوابة ← الحضور | عرض سجل حضورك، تسجيل دخول/خروج |
| طلبات الإجازة | البوابة ← الإجازات | تقديم طلب إجازة |
| الرواتب | البوابة ← الرواتب | عرض قسائم راتبك الشهرية |
| الملف الشخصي | البوابة ← الملف الشخصي | تحديث بياناتك الشخصية واللغة |

### كيف تطلب إجازة

1. البوابة ← الإجازات ← طلب جديد.
2. اختر نوع الإجازة وتاريخ البداية وتاريخ النهاية وأضف ملاحظة.
3. أرسل. سيوافق أو يرفض مدير الموارد البشرية.
4. تصلك إشعار عند تغيير الحالة.

### إجراءات تحتاج موافقة

كل ما يؤثر على الراتب أو الجدول يحتاج موافقة:
- طلبات الإجازة
- طلبات السُّلف (إذا كانت متاحة)

### نصائح

- اللغة: البوابة ← الملف الشخصي ← غيّر اللغة المفضّلة. التطبيق يتبدّل فورًا.
- قسائم الراتب تظهر بعد معالجة الرواتب كل شهر. إذا لم تجد قسيمتك، مدير
  الموارد البشرية لم يُشغّل مسيرة هذا الشهر بعد.
